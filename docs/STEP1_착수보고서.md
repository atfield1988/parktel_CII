## 1. 프로젝트 개요

### 1.1 목적
본 프로젝트는 주요정보통신기반시설 기술적 취약점 분석·평가방법 상세가이드(2026)의 Web Application 점검항목을 기준으로,

3. 취약점별 증적(코드/요청응답/결과화면) 필수 확보
4. 개선 후 재진단(회귀검증) 필수

### 1.3 기준 문서
- 2026 주요정보통신기반시설 기술적 취약점 분석·평가방법 상세가이드

## 4. 판정 기준(확정)

### 4.1 WebApp 21개 항목 판정 기준(가이드라인 발췌 반영)

| 코드 | 항목 | 양호 기준 | 취약 기준 |
|---|---|---|---|
| CI | 코드 인젝션 | 임의 입력값 철저 검증, 허용값만 처리 | 입력 검증 없이 명령 실행 |
| SI | SQL 인젝션 | 비정상 쿼리 실행 차단 검증 적용 | 쿼리 입력 검증 미흡으로 비정상 쿼리 실행 |
| DI | 디렉터리 인덱싱 | 디렉터리 파일 리스트 비노출 | 디렉터리 파일 리스트 노출 |
| EP | 에러 페이지 적용 미흡 | 자체 정의 에러페이지로 과도 정보 비노출 | 기본 에러페이지에 버전/경로/스택 등 노출 |
| IL | 정보 누출 | 중요정보 마스킹, 구조 추정 정보 비노출 | 평문 중요정보/과도한 구조 정보 노출 |
| XS | 크로스사이트 스크립트 | 입력 검증·필터링으로 스크립트 실행 차단 | 검증·필터링 부재로 HTML/스크립트 실행 |
| CF | 크로스사이트 요청 위조(CSRF) | 중요 요청에 CSRF 토큰 적용·검증 정상 | CSRF 토큰 부재 또는 검증 미수행 |
| SF | 서버사이드 요청 위조(SSRF) | 외부 입력 화이트리스트 검증 후 허용 범위만 처리 | 외부 입력 검증 부재로 임의 자원 요청 가능 |
| BF | 약한 비밀번호 정책 | 유추 어려운 비밀번호 또는 높은 복잡도 정책 | 유추 쉬운 비밀번호 또는 낮은 복잡도 정책 |
| IA | 불충분한 인증 절차 | 중요 페이지 추가인증 존재, 서버사이드 우회 불가 | 추가인증 부재 또는 인증 우회 가능 |
| IN | 불충분한 권한 검증 | 중요 페이지 사용자 검증으로 권한 탈취 제한 | 사용자 검증 미흡으로 권한 탈취 가능 |
| PR | 취약한 비밀번호 복구 절차 | 난수 기반 재설정, 인증된 메일/SMS 링크·임시비밀번호 전송 | 일정 패턴 재설정 후 화면 즉시 출력 |
| PV | 프로세스 검증 누락 | 프로세스 검증 존재, 직접접근/변조로 논리오류 미발생 | 검증 미흡으로 논리오류·기능 왜곡·취약점 노출 |
| FU | 악성 파일 업로드 | 업로드 파일 확장자 검증 수행 | 확장자 검증 부재 + 업로드 경로 실행 가능 |
| FD | 파일 다운로드 | 입력 검증, 허용 경로/파일만 접근, 상위경로 차단 | 입력 검증 부재로 임의 경로/비인가 파일 다운로드 |
| IS | 불충분한 세션 관리 | 추측불가 세션ID + 세션 종료시간 설정 | 예측가능 ID 또는 만료 미설정으로 재사용 가능 |
| SN | 데이터 평문 전송 | 중요정보 전송구간 암호화 통신 적용 | 중요정보 전송구간 암호화 미적용 |
| CC | 쿠키 변조 | 서버사이드 세션 또는 안전 알고리즘(SEED/3DES/AES) 적용 쿠키 사용 | 안전 알고리즘 미적용 쿠키 또는 쿠키만으로 인증/권한 부여 |
| AE | 관리자페이지 노출 | 유추 쉬운 URL로 관리자페이지 접근 불가 | 유추 쉬운 URL로 관리자 접근/로그인 가능 |
| AU | 자동화 공격 | 반복 요청 통제가 적절 | 반복 요청 통제가 미흡 |
| WM | 불필요한 Method 악용 | 불필요 메소드 변조 요청 악용 제한 | 불필요 메소드 변조 요청 악용 가능 |

> 기능 미구현 항목은 판정값을 `N/A`로 표기하며, 미구현 검증 증적을 필수 첨부한다.

### 4.2 위험도 표기 원칙
- 심각도 모델(CVSS 등)은 사용하지 않음
- 위험도는 사용자 제공 `위험평가 보고서.xlsx` 모델을 적용
- 보고서에는 **가이드라인 항목 중요도(상/중/하)**와 **산출 위험도**를 함께 표기

### 4.3 위험평가 모델(엑셀 반영)
- 모델 출처: `/home/user/uploaded_files/위험평가 보고서.xlsx`
- 전체 위험도 계산식
  - `위험도C = 자산중요도C + 취약성 + 위협도`
  - `위험도I = 자산중요도I + 취약성 + 위협도`
  - `위험도A = 자산중요도A + 취약성 + 위협도`
  - `전체 위험도 = 위험도C + 위험도I + 위험도A`
- 취약성 계산식
  - `취약성 = 항목중요도 × (2 - (평가가중치 × 2))`
- 평가 입력값 기준
  - 항목중요도: 상=3, 중=2, 하=1
  - 평가가중치: 양호=1, 취약=0, N/A=제외
  - 위협도: STEP3의 PARK-01~PARK-21 위협 목록 값 사용(현재 전 항목 3)
  - 자산중요도(C/I/A): 3/2/1 기준을 자산별로 STEP2에서 확정
- 산출 결과는 STEP4 진단결과표와 STEP6 재진단 결과표에 반영
